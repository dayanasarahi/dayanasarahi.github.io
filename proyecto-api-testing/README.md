# Proyecto QA: Validación de API utilizando Boundary Value Analysis

## Descripción

Este proyecto tuvo como objetivo validar el comportamiento de un endpoint que recibe el parámetro deliveryTime de tipo entero. Se aplicó la técnica de Boundary Value Analysis (BVA) para verificar que el sistema procese correctamente valores válidos e inválidos cercanos a los límites definidos por el requisito.

## Objetivos

* Analizar el requisito funcional.
* Identificar límites válidos e inválidos.
* Diseñar casos de prueba basados en BVA.
* Verificar respuestas de la API.
* Detectar comportamientos inesperados.

QA Engineer Junior

Responsabilidades:

## Técnica aplicada

Boundary Value Analysis (BVA)

Se evaluaron:

* Límite inferior.
* Límite superior.
* Valores inmediatamente inferiores.
* Valores inmediatamente superiores.
* Valores válidos representativos.

## Herramientas utilizadas

* Postman
* JSON
* Google Sheets
* Chrome DevTools

## Escenarios probados

### Casos válidos

* Valores dentro del rango permitido.
* Valores exactamente en los límites.

### Casos inválidos

* Valores menores al mínimo permitido.
* Valores mayores al máximo permitido.
* Datos fuera de especificación.
